Мій перший репозиторій.
1. Що таке Git?
Git - розподілена система контролю версій, яка дає можливість розробникам відстежувати зміни в файлах і працювати спільно з іншими розробниками.
2. В якому статусі можуть перебувати файли при роботі з Git?
 Збережений, змінений, індексований. Збережений у коміті означає, що дані безпечно збережено в локальній базі даних. 
 Змінений означає, що у файл внесено редагування, які ще не збережено в базі даних. 
 Індексований стан виникає тоді, коли ви позначаєте змінений файл у поточній версії,щоб ці зміни ввійшли до наступного знімку коміту.
3. Для чого використовується команда add?
Для того, щоб додати файли у репозиторій.
4. Для чого використовується команда status?
Щоб дізнатися в якому статусі знаходяться файли.
5. Для чого використовується команда commit?
Щоб зберегти зміни у репозиторії локально.
6. Для чого використовується команда init?
Щоб ініціалізувати репозиторій.
7. Що необхідно писати в коментарях комітів?
Вирішення проблеми. Опис змін. Результат коду після змін.
8. Як переглянути список комітів?
За допомогою команди git log.
9. Для чого використовується команда checkout?
Для створення гілок або перемикання між ними.
